# User Stories и Use Cases для TaskFocus

---

## 📝 USER STORIES

### Формат:
**Как** [роль пользователя], **я хочу** [действие], **чтобы** [цель/ценность]

---

### US-001: Создание задачи
**Как** пользователь с СДВГ,  
**Я хочу** быстро создать задачу с минимумом полей,  
**Чтобы** не терять идею и не уходить в перфекционизм оформления.

**Критерии приёмки:**
- [ ] Создание задачи за ≤ 3 клика
- [ ] Обязательные поля только: название
- [ ] Опционально: описание, энергия, дедлайн
- [ ] Автосохранение черновика

---

### US-002: Ограничение активных задач
**Как** пользователь с СДВГ,  
**Я хочу** видеть не более 3 активных задач одновременно,  
**Чтобы** не чувствовать перегрузки и фокусироваться на главном.

**Критерии приёмки:**
- [ ] Система не позволяет активировать > 3 задач
- [ ] При попытке показывается объясняющее сообщение
- [ ] Предлагается завершить или отложить существующую
- [ ] Визуальный индикатор заполненности (1/3, 2/3, 3/3)

---

### US-003: Учёт энергии
**Как** пользователь с СДВГ,  
**Я хочу** помечать задачи требуемой энергией и фильтровать по текущей,  
**Чтобы** делать задачи, которые соответствуют моему состоянию.

**Критерии приёмки:**
- [ ] Шкала энергии: 1-5 или Низкая/Средняя/Высокая
- [ ] Фильтр "Показать задачи для текущей энергии"
- [ ] Задачи сортируются: сначала подходящие по энергии
- [ ] Визуальная индикация: цвет/иконка уровня энергии

---

### US-004: Мягкие дедлайны
**Как** пользователь с СДВГ,  
**Я хочу** указывать диапазон дат вместо точной,  
**Чтобы** снизить тревожность от time blindness.

**Критерии приёмки:**
- [ ] Поле "Сделать с... по..."
- [ ] Возможность указать только одну дату
- [ ] Визуальное отличие от жёстких дедлайнов
- [ ] Напоминание в начале диапазона, не в конце

---

### US-005: Разбиение на подзадачи
**Как** пользователь с СДВГ,  
**Я хочу** разбивать большие задачи на маленькие шаги,  
**Чтобы" преодолевать "стену ужаса" при старте.

**Критерии приёмки:**
- [ ] Добавление подзадачи в 1 клик
- [ ] Подзадачи не учитываются в лимите 3
- [ ] Прогресс-бар выполнения родительской задачи
- [ ] Возможность перетаскивания для сортировки

---

### US-006: Фокус-режим
**Как** пользователь с СДВГ,  
**Я хочу** скрывать всё лишнее при работе над задачей,  
**Чтобы** минимизировать отвлечения.

**Критерии приёмки:**
- [ ] Кнопка "Фокус" на задаче
- [ ] Полноэкранный режим с одной задачей
- [ ] Скрыты: меню, списки, уведомления
- [ ] Таймер/прогресс в фокусе (опционально)

---

### US-007: История и аналитика
**Как** пользователь с СДВГ,  
**Я хочу** видеть, что я уже сделал,  
**Чтобы** чувствовать прогресс и мотивацию.

**Критерии приёмки:**
- [ ] Список завершённых задач
- [ ] Счётчик выполненных задач за день/неделю
- [ ] Визуализация: календарь активности
- [ ] Возможность восстановить случайно закрытую задачу

---

### US-008: Быстрый ввод
**Как** пользователь с СДВГ,  
**Я хочу** добавлять задачи без открытия полной формы,  
**Чтобы" не терять контекст и фокус.

**Критерии приёмки:**
- [ ] Поле быстрого ввода на главном экране
- [ ] Enter = создание задачи
- [ ] Автоматическое определение энергии (по ключевым словам)
- [ ] Голосовой ввод (опционально, MVP-2)

## 🔍 USE CASES (Сценарии использования)

### UC-001: Создание и активация задачи

**Актор:** Пользователь  
**Предусловие:** Приложение открыто  
**Постусловие:** Задача создана и активна (если лимит не превышен)

**Основной сценарий:**
1. Пользователь нажимает "Новая задача"
2. Система открывает диалог создания
3. Пользователь вводит название
4. Пользователь выбирает энергию (опционально)
5. Пользователь указывает дедлайн (опционально)
6. Пользователь нажимает "Создать"
7. Система проверяет лимит активных задач
8. Система создаёт задачу со статусом "Active"
9. Система добавляет запись в историю
10. Система обновляет список задач

**Альтернативный сценарий A (лимит превышен):**
7a. Система обнаруживает 3 активные задачи
8a. Система показывает сообщение: "Лимит достигнут. Завершите или отложите задачу"
9a. Система предлагает список активных задач для действия
10a. Пользователь выбирает: отложить существующую или не создавать новую

**Альтернативный сценарий B (отмена):**
6b. Пользователь нажимает "Отмена"
7b. Система закрывает диалог без сохранения

---

### UC-002: Фильтрация задач по энергии

**Актор:** Пользователь  
**Предусловие:** Есть созданные задачи  
**Постусловие:** Отображены задачи, соответствующие выбранной энергии

**Основной сценарий:**
1. Пользователь оценивает свою текущую энергию (например, 3/5)
2. Пользователь выбирает фильтр "Моя энергия: 3"
3. Система запрашивает задачи пользователя
4. Система фильтрует: EnergyLevel ≤ 3
5. Система сортирует: сначала EnergyLevel = 3, затем ниже
6. Система отображает отфильтрованный список
7. Пользователь выбирает задачу из списка

**Альтернативный сценарий (нет подходящих задач):**
5a. Список отфильтрованных задач пуст
6a. Система показывает сообщение: "Нет задач для этого уровня энергии"
7a. Система предлагает: снизить фильтр или создать простую задачу

---

### UC-003: Разбиение задачи на подзадачи

**Актор:** Пользователь  
**Предусловие:** Есть созданная задача  
**Постусловие:** Созданы подзадачи, связанные с родительской

**Основной сценарий:**
1. Пользователь открывает задачу
2. Пользователь нажимает "Добавить подзадачу"
3. Система показывает поле ввода
4. Пользователь вводит название подзадачи
5. Пользователь нажимает Enter или "Добавить"
6. Система создаёт подзадачу со статусом "Active"
7. Система обновляет прогресс родительской задачи
8. Пользователь повторяет шаги 2-7 для каждой подзадачи

**Основной сценарий (выполнение подзадачи):**
9. Пользователь отмечает подзадачу выполненной
10. Система меняет статус подзадачи на "Completed"
11. Система пересчитывает прогресс родительской задачи
12. Если все подзадачи выполнены → предложить закрыть родительскую

---

### UC-004: Работа в фокус-режиме

**Актор:** Пользователь  
**Предусловие:** Есть активная задача  
**Постусловие:** Пользователь сфокусировался на одной задаче

**Основной сценарий:**
1. Пользователь выбирает задачу из списка
2. Пользователь нажимает "Фокус"
3. Система скрывает: список задач, меню, панели
4. Система показывает полноэкранную карточку задачи
5. Система отображает: название, описание, подзадачи
6. Пользователь работает над задачей
7. Пользователь отмечает подзадачи выполненными
8. Пользователь нажимает "Завершить фокус"
9. Система возвращается к списку задач
10. Система показывает сводку: что сделано

**Альтернативный сценарий (завершение задачи):**
8a. Пользователь нажимает "Задача выполнена"
9a. Система меняет статус на "Completed"
10a. Система показывает поздравление
11a. Система возвращается к списку

---

### UC-005: Обработка просроченной задачи

**Актор:** Пользователь  
**Предусловие:** Есть задача с прошедшим дедлайном  
**Постусловие:** Пользователь принял решение о задаче

**Основной сценарий:**
1. Система обнаруживает просроченную задачу
2. Система помечает задачу визуально (красный цвет/иконка)
3. Пользователь видит пометку
4. Пользователь открывает задачу
5. Система показывает варианты:
   - Продлить дедлайн
   - Разбить на подзадачи
   - Отложить (архивировать)
   - Отметить выполненной
6. Пользователь выбирает действие
7. Система выполняет действие
8. Система убирает пометку просрочки

**Важно:** Никаких штрафов, guilt-tripping или негативных сообщений!

## 🎯 ПРИОРИТЕЗАЦИЯ MoSCoW

### Must Have (MVP — обязательно)
- ✅ Создание/редактирование/удаление задач
- ✅ Ограничение в 3 активные задачи
- ✅ Учёт энергии (1-5)
- ✅ Фильтрация по энергии
- ✅ Мягкие дедлайны
- ✅ Разбиение на подзадачи
- ✅ История выполнения

### Should Have (Желательно)
- 🟡 Фокус-режим
- 🟡 Быстрый ввод
- 🟡 Drag-and-drop сортировка
- 🟡 Теги/категории

### Could Have (Возможно)
- 🔵 Авторизация/регистрация
- 🔵 Синхронизация
- 🔵 Аналитика (графики)
- 🔵 Экспорт данных

### Won't Have (Пока нет)
- ❌ AI-рекомендации
- ❌ Body doubling (видео)
- ❌ Мобильное приложение
- ❌ Интеграция с календарями

## 💬 СЦЕНАРИИ ДЛЯ UX-ТЕСТИРОВАНИЯ

### Сценарий 1: Первое знакомство (15 минут)
**Цель:** Оценить простоту освоения

1. Откройте приложение
2. Создайте 3 задачи на сегодня
3. Укажите энергию для каждой
4. Отфильтруйте по своей текущей энергии
5. Отметьте одну задачу выполненной

**Вопросы после:**
- Сколько времени заняло освоение?
- Что было непонятно?
- Что хотелось бы улучшить?

---

### Сценарий 2: Управление перегрузкой (10 минут)
**Цель:** Проверить работу лимита в 3 задачи

1. У вас уже есть 3 активные задачи
2. Попробуйте создать четвёртую
3. Прочитайте сообщение системы
4. Выберите, что делать с существующими задачами

**Вопросы после:**
- Понятно ли сообщение?
- Помогает ли это сфокусироваться?
- Что вы чувствовали при ограничении?

---

### Сценарий 3: Сложная задача (15 минут)
**Цель:** Проверить разбиение на подзадачи

1. Создайте задачу "Подготовить презентацию"
2. Разбейте её на 5-7 подзадач
3. Выполните 3 подзадачи
4. Посмотрите прогресс

**Вопросы после:**
- Помогло ли разбиение начать?
- Удобно ли управлять подзадачами?
- Мотивирует ли прогресс-бар?

## 📊 МЕТРИКИ УСПЕХА

### Количественные:
- Время создания задачи (цель: < 30 сек)
- Время освоения (цель: < 15 мин)
- Количество выполненных задач/день
- Процент просроченных задач

### Качественные:
- Субъективная оценка перегрузки (1-5)
- Ощущение контроля над задачами
- Готовность рекомендовать другу
