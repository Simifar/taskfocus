# Контекст дипломной работы

Этот документ помогает связать программный проект TaskFocus с текстом дипломной работы.

## Рабочая тема

Возможная формулировка:

> Разработка веб-приложения для планирования задач с учетом когнитивной нагрузки пользователя.

Более прикладная формулировка:

> Разработка веб-приложения TaskFocus для управления задачами и поддержки концентрации.

## Актуальность

Многие приложения для управления задачами ориентированы на универсальную продуктивность: списки, дедлайны, приоритеты, календарь. Для пользователей с проблемами концентрации такой подход может создавать дополнительную нагрузку: длинные списки задач, жесткие сроки и постоянный выбор между большим количеством дел усиливают прокрастинацию и тревожность.

TaskFocus исследует более мягкий подход к планированию:

- ограничение количества активных задач;
- учет субъективного уровня энергии;
- мягкие дедлайны как диапазоны дат;
- декомпозиция задач на подзадачи;
- отдельная зона входящих задач;
- несколько представлений для планирования.

## Объект и предмет

Объект исследования:

> Процесс персонального планирования задач в веб-приложениях.

Предмет исследования:

> Методы и программные средства организации задач с учетом уровня энергии пользователя, мягких дедлайнов и ограничения фокуса.

## Цель

Цель работы:

> Спроектировать и разработать веб-приложение для управления задачами, которое снижает когнитивную перегрузку за счет ограничения активного фокуса, учета уровня энергии и использования мягких дедлайнов.

## Задачи

1. Проанализировать предметную область и существующие решения для управления задачами.
2. Сформулировать требования к приложению.
3. Спроектировать архитектуру full-stack веб-приложения.
4. Разработать модель данных и серверный API.
5. Реализовать клиентский интерфейс для планирования задач.
6. Реализовать авторизацию пользователей.
7. Реализовать бизнес-правила ограничения фокуса и мягких дедлайнов.
8. Провести тестирование и оценить результат.

## Функциональные требования

- Пользователь может зарегистрироваться и войти в систему.
- Пользователь может войти через Google OAuth.
- Пользователь может создавать, редактировать, выполнять, архивировать и удалять задачи.
- Пользователь может создавать подзадачи.
- Пользователь может задавать приоритет задачи.
- Пользователь может задавать уровень энергии задачи от `1` до `5`.
- Пользователь может задавать мягкий дедлайн как диапазон дат.
- Система ограничивает количество активных задач на сегодня.
- Пользователь может просматривать задачи в разделах: сегодня, входящие, неделя, календарь, день, архив.
- Пользователь может видеть статистику по задачам.

## Нефункциональные требования

- Интерфейс должен быть русскоязычным.
- Приложение должно работать в браузере без установки дополнительного ПО.
- Данные пользователя должны быть изолированы от данных других пользователей.
- API должно проверять авторизацию на сервере.
- Пароли должны храниться в хешированном виде.
- Приложение должно поддерживать деплой на Vercel и PostgreSQL/Neon.
- Код должен быть типизирован через TypeScript.

## Ограничения текущей версии

- Нет автоматических тестов.
- Dashboard преимущественно client-side.
- Auth flow совмещает custom JWT и NextAuth.
- Восстановление пароля сейчас реализовано как UI-заглушка.
- Нет командной работы и совместных проектов.

Эти ограничения можно честно указать в дипломе как область дальнейшего развития.
